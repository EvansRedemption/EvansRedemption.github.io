消息通知对比

发表于 2019-03-23 | 分类于 交互面试题试答

在社交软件中,如Facebook、twitter、Google+等，通常将消息通知放在底部标签栏。并且通常在浏览的主界面还有一些交互补偿，如出现“你有新动态”的提示等；而在社交通讯类软件中，如微信、what’s app、 line等 ，消息通知则显示在页面列表内。请分析他们可能的背景和设计想要达成的目标，并对比消息通知的内容类型及设计的优缺点。

背景

Facebook、Twitter、Google+等应该归类为SNS
WeChat、What’s App、Line等应该归类为IM Tool

我认为社交类产品最终目的一定是在网络中还原某个现实社交场景，举个例子
SNS是一个广场，我们发送任何信息都等于在这个广场上大喊一声“xxxxxxxx”，感兴趣的人就来找你，围成一圈讨论balabala。
IM Tool则是你家，你作为房主，邀请某个人/某群人进屋子聊，不是所有人都能看到。

那么，从背景角度就很容易理解SNS和IM Tool不同的消息通知形式。SNS消息通知是一块广告牌，所有人都能见。IM Tool则是指名道姓的某某某，我和你说“xxxxx”。
所以SNS的通知放在底部标签栏，IM Tool放在页面列表内。

设计目标

我们首先考虑两种类型社交产品的存在意义、目的

  • IM Tool是线下关系链在线上的一种映射，承载的主要（唯一？）功能是维持线下熟人关系链。本质是工具，用完即走。
  • SNS是获取群体存在感、认同感，用以维护人际关系。注意，这里的人际关系应该理解为广义人际，即包含不熟悉的人。本质是平台，用于广播。

那么从设计目的角度考虑也就很容易去理解为什么这么设计了。
IM Tool目标对象是一个具体的用户，使用目的是和Ta聊天，那么消息通知就应该限定在这个特定的用户中，也就是页面列表内。
SNS目标对象是所有关系链（此处不讨论类似朋友圈之类的非公开SNS），不存在某个特定的对象接收信息，他是一个全平台广播，所有人都能接收到。真要较真去提@功能，那也是一个全平台广播，只是被@的对象获得一个独有的强提醒罢了。

消息通知

消息通知本质理解就是传达系统，承接需求，启发后续操作。
消息通知大抵可以分为三类

  • 公告：针对全部用户的消息推送
  • 提醒：指在特定环境下的通知
  • 私信：点对点聊天

题中的SNS消息通知应该是指提醒，即“有人更新了流状态”等等，IM Tool消息通知是指私信类型。
无论是哪种类型的消息通知，都应该存在两个对象，即需求方和接收方。
