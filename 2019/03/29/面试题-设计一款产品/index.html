面试题-设计一款产品

发表于 2019-03-29 | 分类于 交互面试题试答

选择旅游、商品、理财、食品、体育其中一类，设计一个你觉得有需求且目前没有得到较好满足的功能，可以是一款小而美的产品，也可以是承载在某一款产品上的功能

前言

这款产品其实我已经设想很久了，借这个机会阐述、归拢一下。后期我应该会通过Fake Project的形式去做出来，那时候会有更详细的内容。

设计背景

我出游习惯先做攻略，而且绝大多数攻略会详细到具体时间段，比如

09:00-10:00 起床吃早饭，在xx地方有个xx店，可点xx。步行过去xx分钟，路线xx
10:00-12:00 去xx地方，玩xx，午饭在附近的xx解决

这样的计划表很舒服，吃喝玩乐行都安排的妥妥当当对吧。
但是，在制作攻略的时候简直就是噩梦。
首先你得根据一些信息，确定自己的必玩点，必吃点。
然后，再根据这些点位的分布和相应的时间节点（吃饭，睡觉）去确定自己的住宿、交通。然后再根据景点、食宿点之间的距离，交通情况、预算等等一大堆内容去反复权衡。
这些还只是概念，当你实际操作的时候。你打开百度地图，只能选择任意两个点位查询交通和时间花销。如果你有7,8个点位呢，你需要查多少次？需要交叉对比多少次？而且万一时间不合适呢？全盘重来！
还要去各种点评网站上查看你选的地方合不合适，好不好吃，好不好玩。头疼吗？
噩梦吧，我宁愿无攻略出行 : ）

当你落地了，开始你的旅程了。又会因为交通等车、游玩时间不定等等特殊情况导致计划失效，不尴不尬的剩下一半时间，玩也不是不玩也不是，你开心吗？
我选择死亡 ：）

问题陈述

其实就像我在背景中陈述的，我希望能够有一个智能的工具来安排我的旅行计划，如何去达成这个目的即为这个项目的核心问题。

设计目标

设计一款产品，能够通过筛选等方式，一键生成路径并标注时间、交通、评价等重点信息。

用户访谈

略，暂时只是个概述，真的去实施这个Fake Project的时候，会把这些补上。同时，会修正设计目标、设计机会点等…现在只是单纯的根据自我认知去描述这个项目。

故事版/User Journey and etc.

略，暂时只是个概述，真的去实施这个Fake Project的时候，会把这些补上。同时，会修正设计目标、设计机会点等…现在只是单纯的根据自我认知去描述这个项目。

设计机会点

因为是个Fake Project，暂时不讨论商业层面的问题。

  • 信息聚合。聚合各类评价网站对于某个点位的评价。包括且不限于景点、网红店、食宿点。给用户提供有价值的的点位选择。
  • 路径规划。用户挑选点位后，可根据各类限制条件（游玩时间…）自动规划合适的路径
  • 智能推荐。用户生成路径后，可以智能推荐更优的点位选择，并生成对比图。
  • 路径分享。可以通过路径的分享，形成一个出行设计，这可能是后期的一个方向。

后记

其实这个问题的核心是希望你阐述清楚你想要一个什么样的产品，它的出发点是什么？解决了什么问题？你是如何解决的？
我认为我并没有回答清楚，有点好高骛远的想要直接产出一个Presentation，但实际上现在手头的信息并不能支撑这一点，我在考虑是不是重新写。按照面试情况写，而不是按照Fake Project

还是思路有问题。如何去设计一个产品（0-1）的思路存在很大的问题，脑子里有很多节点，但是这些节点不能够很好的串联起来，导致叙述上有点前后矛盾，也导致自己在实际操作中会忽略很多重要的设计节点。
当反思，亦当改正。
